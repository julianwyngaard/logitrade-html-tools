# Julian's HTML Tools

A collection of single-file HTML tools hosted at [tools.julianwyngaard.dev](https://tools.julianwyngaard.dev).

Inspired by [Simon Willison's tools](https://tools.simonwillison.net/) and his [blog post on building HTML tools](https://simonwillison.net/2025/Dec/10/html-tools/).

## Philosophy

Each tool is a **standalone HTML file** with inline CSS and JavaScript. This means:

- **No build step** for the tools themselves
- **Copy-paste deployable** - each tool works independently
- **CDN dependencies** - external libraries loaded from CDNs when needed
- **Client-side only** - no server required, all processing happens in the browser
- **URL state** - tools can store state in URLs for sharing

## Tools

### Data & Text

- **[JSON Formatter](json-formatter.html)** - Format, validate, and minify JSON. Shareable via URL.

## Quick Start

```bash
# Clone the repo
git clone https://github.com/julianwyngaard/logitrade-html-tools.git
cd logitrade-html-tools

# Build the index and colophon pages
npm run build

# Serve locally for testing
npm run serve
# Then open http://localhost:3000
```

No dependencies to install - the build scripts use only Node.js built-in modules.

## Adding a New Tool

### Step 1: Create the HTML file

Create `[tool-name].html` in the root directory. Use this template:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>[Tool Name] - tools.julianwyngaard.dev/</title>
    <style>
        * { box-sizing: border-box; }
        body {
            font-family: system-ui, -apple-system, sans-serif;
            max-width: 900px;
            margin: 0 auto;
            padding: 1rem;
            line-height: 1.6;
            color: #333;
        }
        /* Add your tool-specific styles */
    </style>
</head>
<body>
    <a href="index.html" class="back-link">&larr; All Tools</a>
    <h1>[Tool Name]</h1>
    <p>[One line instruction to the user on using the tool]</p>

    <!-- Your tool UI here -->

    <footer>
        <a href="index.html">tools.julianwyngaard.dev</a> |
        <a href="https://github.com/julianwyngaard/logitrade-html-tools">Source</a>
    </footer>

    <script>
        // Your tool logic here
    </script>
</body>
</html>
```

### Step 2: Create documentation

Create `[tool-name].docs.md` with this structure:

```markdown
# [Tool Name]

Brief description of what the tool does.

## Features

- **Feature 1**: Description
- **Feature 2**: Description

## Usage

1. Step one
2. Step two

## URL Parameters

- `?param=value` - What this parameter does

## Technical Notes

- Implementation details users should know
- Privacy/security notes (e.g., "all processing happens client-side")

## Created

YYYY-MM-DD - Initial implementation
```

### Step 3: Add to tools.json

Add an entry to `tools.json`:

```json
{
  "slug": "tool-name",
  "name": "Tool Name",
  "description": "Brief description for the index page.",
  "category": "data",
  "created": "2026-01-18",
  "updated": "2026-01-18"
}
```

Available categories: `data`, `image`, `dev`, `misc` (or add new ones - see below).

### Step 4: Build and test

```bash
npm run build    # Regenerates index.html and colophon.html
npm run serve    # Test locally
```

### Step 5: Commit

```bash
git add tool-name.html tool-name.docs.md tools.json index.html colophon.html
git commit -m "feature: add [tool-name] tool"
git push
```

## Build System

The build system generates two pages from `tools.json`:

### index.html

The main tool listing page, organized by category. Generated by `scripts/build-index.js`.

### colophon.html

A "behind the scenes" page showing each tool's description (from `.docs.md` files) and git commit history. Generated by `scripts/build-colophon.js`.

### npm scripts

| Command | Description |
|---------|-------------|
| `npm run build` | Run all build scripts |
| `npm run build:index` | Regenerate only index.html |
| `npm run build:colophon` | Regenerate only colophon.html |
| `npm run serve` | Start local dev server (uses npx serve) |

## tools.json Reference

```json
{
  "tools": [
    {
      "slug": "tool-name",        // URL-safe identifier (matches filename)
      "name": "Tool Name",        // Display name
      "description": "...",       // Short description for index
      "category": "data",         // Category key
      "created": "2026-01-18",    // ISO date
      "updated": "2026-01-18"     // ISO date (update when tool changes)
    }
  ],
  "categories": {
    "data": {
      "name": "Data & Text",      // Display name
      "order": 1                  // Sort order (lower = first)
    },
    "image": {
      "name": "Image & Media",
      "order": 2
    },
    "dev": {
      "name": "Development",
      "order": 3
    },
    "misc": {
      "name": "Miscellaneous",
      "order": 99
    }
  }
}
```

To add a new category, just add it to the `categories` object with a unique key, display name, and order.

## Deployment

Deployment is automatic via GitHub Actions:

1. Push to `main` branch
2. GitHub Actions runs `npm run build`
3. Generated files are deployed to GitHub Pages

The workflow uses `fetch-depth: 0` to get full git history, which the colophon generator needs for commit history.

## Design Principles

Based on [Simon Willison's guidance](https://simonwillison.net/2025/Dec/10/html-tools/):

1. **Single-file tools** - Each tool is one HTML file with inline CSS/JS
2. **No build step for tools** - Tools work by just opening the HTML file
3. **CDN for dependencies** - Use CDN links instead of npm packages
4. **URL state** - Store shareable state in URL parameters
5. **localStorage for secrets** - Keep API keys client-side only
6. **Client-side processing** - No server needed, privacy-preserving
7. **Keep it small** - Aim for a few hundred lines per tool
